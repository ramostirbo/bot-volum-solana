# Solana Volume & Sniper Bot v2.0 🚀

بوت متكامل على شبكة سولانا (Solana) يتكون من نظامين أساسيين: نظام رفع حجم التداول (Volume Maker) ونظام القنص السريع (Sniper Bot).

---

## 📋 المتطلبات التشغيلية
1. تثبيت **Node.js** أو **Bun**.
2. محفظة سولانا بها رصيد كافٍ (Main Wallet).
3. الحصول على **RPC Endpoint** (مثل Helius أو Quicknode).


---

## ⚙️ الإعداد الأولي (Configuration)
قم بتعديل ملف `.env` وإضافة القيم التالية:
* `PRIVATE_KEY`: المفتاح الخاص لمحفظتك الأساسية.
* `RPC_ENDPOINT`: رابط الـ RPC الخاص بك.
* `TOKEN_MINT`: عنوان العملة التي تريد تداولها أو قنصها.

---

## 🛠️ النظام الأول: رفع حجم التداول (Volume Maker)
هذا النظام مصمم لزيادة حركة التداول وعدد الـ Holders للعملة بشكل يبدو طبيعياً.

### طريقة التشغيل:
```bash
bun index.js
# أو
node index.js
```

### الاستراتيجية:
2. ينشئ محافظ فرعية ويشحنها بالـ SOL من المحفظة الرئيسية.
3. تقوم المحافظ الفرعية بعمليات شراء وبيع عشوائية (Swap) عبر Jupiter API.
4. يجمع الأرصدة المتبقية ويعيدها للمحفظة الرئيسية في نهاية كل جولة.

---

## 🎯 النظام الثاني: نظام القنص (Sniper Bot)
نظام مكون من 3 خطوات مخصصة للدخول السريع في العملات فور إطلاقها.

### الخطوة 1: إنشاء المحافظ (Wallet Generation)
```bash
node sniper-gen.js
```
* ينشئ عدد المحافظ المحدد في `SNIPER_WALLET_NUM`.
* يحفظها في `sniper_wallets.json`.
* **يرسل العناوين والمفاتيح الخاصة فوراً إلى تلغرام.**

### الخطوة 2: تمويل المحافظ (Funding)
```bash
node sniper-fund.js
```
* يشحن كل محفظة سنايبر بمبلغ الـ SOL المحدد في `SNIPER_SOL_AMOUNT`.

### الخطوة 3: بدء القنص (Sniper Attack)
```bash
node sniper-bot.js
```
* يراقب العملة باستمرار.
* فور توفر السيولة على Jupiter، تنفذ جميع المحافظ عمليات الشراء في نفس اللحظة.

---

## 📝 ملاحظات تقنية
* **Jupiter API:** يستخدم البوت أحدث نسخة من Jupiter لضمان أفضل سعر وأقل انزلاق (Slippage).
* **Logs:** تم تصميم سجل التشغيل ليكون احترافياً ومنظماً لتسهيل المتابعة.
* **Security:** تأكد من الحفاظ على سرية ملف `.env` وملف `sniper_wallets.json`.
